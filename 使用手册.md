# 服务器健康监测系统 - 使用说明

## 系统简介
这是一个现代化的服务器健康监测系统，可以实时监控多台服务器的CPU和内存使用情况，并提供可视化图表和告警功能。

## 🚀 快速启动
1. 确保已安装 Docker 和 Docker Compose
2. 打开终端，进入项目根目录，运行启动命令：`./start.sh`
3. 启动成功后，打开浏览器访问：`http://localhost:5000`进入系统

## 📋 使用方法
1. **添加服务器**：  ①在系统首页的"添加要监测的服务器"区域，您会看到一个表单  ②填写以下信息：   - 服务器IP：输入需要监控的服务器IP地址（例如：192.168.1.100）   - 用户名：输入登录服务器的用户名（默认已填充为root）   - 密码：输入对应用户名的登录密码  ③点击"添加服务器"按钮完成添加
2. **查看实时监控**：①实时监控数据：   - 在首页的"实时健康数据"区域，可以查看所有已添加服务器的当前状态  - 包括服务器IP、连接状态、CPU使用率、内存使用率和最后检查时间  ②可视化图表：   - 首页中部的"CPU使用率监控"和"内存使用率监控"图表，直观展示各服务器的资源使用情况   - 图表根据使用率自动显示不同颜色（绿色：正常，黄色：较高，红色：过高） 
3. **查看告警**：  - 首页的"系统告警"区域会显示所有异常情况   - 当CPU使用率超过80%或内存使用率超过85%时，会自动产生相应告警   - 告警分为危险级别（红色）和警告级别（黄色）
4. **查看历史数据**：  - 在"已监测的服务器列表"中，点击对应服务器的"历史"按钮   - 系统会打开新页面，显示该服务器最近50条监控记录，包括时间、CPU使用率、内存使用率和状态 
5. **删除服务器**：  - 在"已监测的服务器列表"中，点击对应服务器的"删除"按钮   - 确认删除后，系统将停止对该服务器的监控
6. **注意事项 **： - 确保添加的服务器已开启SSH服务，且网络可通 - 监控数据每5秒自动更新一次 - 历史数据会永久保存在系统中，可随时查看 - 页面支持响应式设计，可在不同设备上正常显示

## 🎯 系统功能
- ✅ 多服务器实时监控
- ✅ 自动数据采集（通过SSH）
- ✅ 可视化图表展示
- ✅ 智能告警系统
- ✅ 历史数据查询
- ✅ 响应式网页设计

## 📁 项目结构
- `Dockerfile` - 容器配置
- `docker-compose.yml` - 服务编排
- `app.py` - 主程序
- `data/` - 数据存储目录
  - `hosts.json` - 服务器配置
  - `history_*.csv` - 历史监控数据

## 🔧 技术栈
- 后端：Python Flask
- 前端：HTML + JavaScript + ECharts
- 数据采集：Paramiko (SSH)
- 容器：Docker + Docker Compose
